EventLoop

任务队列

同步任务优先于异步任务。

  • 在同步队列被执行完之前，任何异步的操作都不会被执行
  • 同步任务执行完之后才执行异步任务

运行机制

for(vari=0;i<4;i++)<
setTimeout(function () [
console.log(i);
), 1000);

涉及异步队列涉及的时间，浏览器发现settimeout不会马上放到运行栈里去，for循环体执行完才去执行异步队列，异步队列没有到时间去收编settimeout，再等待事件循环里处理

Event Loop事件循环

  1. 运行栈运行的是同步任务。

  2. 浏览器识别了事件是一个异步任务，他不会把他放在运行栈里，而是拿走。

  3. 拿走了之后也不是立马放在异步队列中，同步任务执行完以后，再放入异步队列。

  4. 运行栈里没有任务了，就开始执行异步任务。

  5. 执行完后如果运行栈里没有任务了，再监听异步队列。

  6. 这个循环的过程就是EventLoop

异步任务 setTimeout 和setInterval DOM事件 Promise

for (var i = 0; i < 4; i++) {
    setTimeout(function() {
        console.log(i);
    }, 1000)
    console.log('one:' + i);
    setTimeout(function() {
        console.log(i + 1);
    }, 0)
    console.log('two:' + (i + 1));
}
